# Learning Roadmap

| Stage | Goal | Deliverable |
|---|---|---|
| 1 Foundation | Syntax, types, control flow, functions, debugging basics | Four tiny CLI programs |
| 2 Structured code | Modules, files, tests, naming, errors, Git basics | One tested module per language |
| 3 System view | Memory, databases, APIs, async flows, state machines, builds, logging, security | Parser + API + script demo |
| 4 Productive applications | Discord bot, Flask API, embedded parser, Lua events, C++ gameplay core, CI/CD | Four reference projects |
| 5 Advanced architecture | Cross-language contracts, agents, RAG, profiling, Unreal-style architecture | Integrated reference project |

## Weekly plan

- Week 1: Python and Lua foundations.
- Week 2: C and C++ foundations with memory/lifetime focus.
- Week 3: Testing, logging, error handling, and project structure.
- Week 4: APIs, databases, state machines, and event systems.
- Week 5: Security, CI/CD, deployment, and dependency checks.
- Week 6: Cross-language contracts and final integrated project.

A topic is mastered when you can explain it, implement it, test edge cases, debug failures, and document trade-offs.
