# Mission and Scope

## Mission

Create a durable engineering knowledge base that helps a coding agent and human learner design, implement, test, and review production-quality software in Python, C, Lua, and C++.

## Scope

Included: language foundations, real project patterns, secure coding, testing, architecture decisions, cross-language boundaries, exercises, quality gates, and source policy.

Out of scope: storing secrets, pretending examples were executed, deploying production systems, or replacing official standards.

## Design principles

| Principle | Rule |
|---|---|
| Architecture first | Define data flow, ownership, and failure modes before code. |
| Security by default | Validate inputs, protect secrets, log safely, and fail predictably. |
| RAG-friendly | One topic per file, metadata, clear headings, links, and keywords. |
| Production humility | Mark unexecuted examples as NOT VERIFIED. |
