# Definition of Done

A module is done only when:

- Requirements, assumptions, data flow, failure modes, and security boundaries are documented.
- Inputs are validated and errors are explicit.
- No secrets are stored in code or documentation.
- Code examples use English identifiers and comments.
- Tests or verification commands are listed accurately.
- NOT VERIFIED is used whenever an example was not executed.
- Cross-language code documents ownership, lifetime, encoding, and error translation.
- Maintenance tasks and known limits are recorded.
